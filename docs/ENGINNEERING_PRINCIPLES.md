# Syntrix Engineering Principles

## 1. Components have one responsibility.

## 2. Pages assemble components.

## 3. Data lives outside the UI.

## 4. Business logic never belongs inside components.

## 5. Duplicate code is refactored.

## 6. Every commit should move the product forward.

## 7. Build for clarity before optimization.

## 8. If a component becomes difficult to understand, split it.

## 9. Design first. Code second.

## 10. The product philosophy always wins.

## 11. Keep files honest.
A file should only import what it actually uses.

## 12. AI recommendations must be traceable to data.
